

# Dimension Shift Ultimate ◈

Двухмерный пазл-платформер с открытым исходным кодом, разработанный на языке **Python 3.12+** с использованием библиотеки **Pygame 2.0+**. 

Ключевая особенность проекта — механика пространственного дуализма (мгновенного фазирования персонажа между двумя независимыми измерениями) и полностью автономная архитектура: игра функционирует без использования внешних графических и аудио-ассетов, генерируя весь контент процедурно в реальном времени.

---

## 🎮 Архитектура игрового процесса и механика дуализма

Игровое пространство состоит из двух альтернативных измерений, сосуществующих на одной координатной сетке:
* **Измерение А («Cyber»):** Техногенное неоновое пространство. Визуальный образ формируется жесткими геометрическими формами, контрастными линиями цвета Cyan `RGB(0, 255, 255)` на глубоком темном фоне `RGB(12, 12, 28)`.
* **Измерение B («Fantasy»):** Органическое пространство мистического леса. Доминируют мягкие природные формы, округлые силуэты растительности, теплые тона цвета Gold `RGB(255, 200, 0)` на фоне сумеречного неба `RGB(120, 190, 230)`.

**Фазирование объектов:** Препятствия, платформы и враги, физически материальные в текущем измерении, в альтернативном мире переводятся в состояние «призраков» (ghost-объектов) с альфа-каналом $\alpha \in [35, 45]$. Это дает игроку визуальную подсказку о геометрии альтернативного пространства, позволяя планировать траектории движения непосредственно в воздухе.

## 🚀 Ключевые технические фичи и математический аппарат

### 1. Физический движок и инерциальные системы
* **Численное интегрирование:** Расчет перемещения игрока по осям $X$ и $Y$ реализован по методу Эйлера с учетом векторов ускорения, сил сопротивления, силы тяжести ($g = 0.45$ пиксель/кадр²) и терминальной скорости падения ($v_{terminal} = 16.0$ пикселей/кадр).
* **Инерция платформ:** При нахождении на движущихся платформах (`MovingPlatform`) игрок наследует их скорость. При прыжке с движущейся платформы горизонтальный импульс платформы сохраняется и плавно угасает в воздухе с коэффициентом затухания $k = 0.95$.
* **Улучшенный игровой отклик:** Интегрированы алгоритмы *Coyote Time* (активное окно в 8 кадров для прыжка после схода с платформы) и *Jump Buffering* (буферизация ввода прыжка за 8 кадров до касания земли).

### 2. Процедурный синтез аудио (DSP на лету)
* **Прямой синтез в ОЗУ:** Звуковые эффекты генерируются математически в виде 16-битных массивов PCM-данных. Например, частота сигнала прыжка линейно нарастает по формуле:
  $$f(t) = 150 + \left(\frac{t}{D}\right) \cdot 480$$
  где $D = 0.15$ с (длительность звука).
* **Плавный кроссфейд (Дискретный фильтр НЧ):** При смене миров музыкальные треки плавно перетекают друг в друга. Вместо тригонометрического панорамирования в коде используется асимптотическая экспоненциальная интерполяция на каждом кадре с шагом $k = 0.04$, описываемая разностным уравнением:
  $$V_{t+1} = V_t + (V_{target} - V_t) \cdot 0.04$$
  что физически эквивалентно сглаживающему аналоговому фильтру низких частот первого порядка.

### 3. Отказоустойчивое сохранение данных (Anti-Corruption & Anti-Cheat)
* **Атомарная запись на диск:** Чтобы исключить повреждение файлов сохранений при внезапном закрытии процесса или отключении питания, запись производится сначала во временный файл `highscore.json.tmp`, после чего вызывается системный вызов `os.replace()`, атомарно перезаписывающий оригинальный файл.
* **Автоматическое восстановление:** Интегрирована система создания зеркальной резервной копии (`highscore.bak`). При обнаружении повреждения структуры JSON при запуске, игра автоматически восстанавливает прогресс из бэкапа.
* **Обфускация:** Сериализованная строка перед записью кодируется алгоритмом Base64, предотвращая редактирование файла рекордов «в лоб».

## 🧠 Технические вызовы и оптимизация (С чем пришлось столкнуться)

При разработке игры на Pygame был решен ряд сложных инженерных задач, связанных с ограничениями производительности интерпретатора Python и особенностями рендеринга на CPU:

### 1. Борьба с утечками памяти и фризами Garbage Collector (Сборщика мусора)
* **Проблема:** Постоянное создание поверхностей (`pygame.Surface`) с альфа-каналом в игровом цикле для отрисовки системы частиц, мягкого радиального свечения (`get_glow_surf`), шлейфа игрока и текста приводило к лавинообразному росту потребления ОЗУ и регулярным «фризам» физики (микролагам) из-за принудительного запуска сборщика мусора Python.
* **Решение:** 
  1. Реализован строгий механизм кэширования текстур свечения (`_GLOW_CACHE`) и системных шрифтов (`_FONT_CACHE`).
  2. Все тяжелые вспомогательные альфа-поверхности (шлейфы `_trail_surf_cyber`, сканирующие линии CRT, оверлеи) предвыделяются в памяти (преаллокация) при инициализации классов и очищаются через `.fill((0, 0, 0, 0))` вместо повторного инстанцирования.

### 2. Предотвращение застревания в геометрии и скольжение по углам
* **Проблема 1 (Застревание при сдвиге):** При мгновенной смене мира игрок мог оказаться в пространстве, где в альтернативном мире сгенерирована твердая платформа.
* **Решение 1:** Внедрен алгоритм прогнозирования коллизий. Метод `switch_dimension` создает гипотетическую маску игрока в целевом измерении и проверяет пересечения. Если обнаружено застревание в твердом теле или опасности — телепортация блокируется.
* **Проблема 2 (Зацепление за углы):** При прыжке вверх игрок часто задевал платформу краем хитбокса (буквально на 1-2 пикселя), что приводило к мгновенной остановке вертикального импульса.
* **Решение 2:** Реализован алгоритм «соскальзывания» с углов. Если перекрытие хитбокса игрока и угла платформы составляет менее 10 пикселей, движок автоматически смещает вещественную координату игрока `self.fx` в сторону свободного пространства, позволяя проскочить мимо угла без жесткого столкновения.

### 3. Интегрирование субпиксельного движения
* **Проблема:** При округлении координат до целых чисел (типично для хитбоксов Pygame) малые скорости движения (например, плавное затухание скорости платформы в воздухе или движение на низких скоростях) приводили к дерганью картинки или полной остановке движения.
* **Решение:** Все физические расчеты переведены в вещественные координаты типа `float` (`self.fx`, `self.fy`). Целочисленные хитбоксы `self.rect` обновляются только на этапе проверки столкновений и рендеринга путем отсечения дробной части.

## 📦 Запуск и компиляция

### Вариант 1. Запуск скомпилированной сборки (Windows)
Проект скомпилирован в единый исполняемый `.exe` файл с помощью PyInstaller, упаковавший интерпретатор и библиотеки внутрь исполняемого файла:
1. Перейдите в раздел **Releases** на GitHub.
2. Скачайте архив `DimensionShift.zip`.
3. Распакуйте архив и запустите `DimensionShift.exe`.

### Вариант 2. Запуск из исходного кода (Multiplatform)
Для запуска из исходного кода требуется Python версии 3.10+:

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Ventilyator2805/dimension-shift--game.git
   cd dimension-shift--game

2.  Установите библиотеку Pygame:
    pip install pygame
3.  Запустите главный игровой скрипт:
    python main.py
